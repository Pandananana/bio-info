Background

In this assignment you will analyse spatial single-cell transcriptomic data generated with the CosMx Spatial Molecular Imager (NanoString/Bruker) from a mouse brain tissue section (coronal hemisphere).

Spatial transcriptomics enables the simultaneous measurement of gene expression and physical location of cells within a tissue section. Unlike bulk or dissociated single-cell RNA-seq, it preserves the spatial context of transcriptional heterogeneity — allowing us to ask not only which cell types are present, but where they reside within the tissue.

The CosMx platform preserves the spatial position of each cell within the tissue, enabling not only the identification of cell types but also the study of how they are physically organised within anatomical structures.

The dataset

The dataset uses a targeted CosMx Mouse Neuroscience RNA panel of ~950 genes measured across thousands of cells. Because the panel is already focused on biologically relevant targets, we deliberately do not perform Highly Variable Gene (HVG) selection before dimensionality reduction — filtering genes at this stage would risk discarding signal from rare cell populations.

CosMx data are generated using an imaging-based approach where RNA molecules are detected in situ via sequential fluorescent probe hybridisation cycles. Each cell retains its spatial \((x, y)\) position in the tissue throughout.

Dimensionality reduction

PCA

Because the CosMx panel is already targeted (~950 genes), we run PCA on all genes — no HVG selection is applied.

Why scale? Without scaling, highly expressed genes dominate the top PCs simply because they have larger absolute variance. ScaleData() centres each gene to mean 0 and scales to variance 1, ensuring every gene contributes equally to PCA.

# Make normalised counts available to Seurat's PCA (no re-normalisation)
obj[["RNA"]]$data <- obj[["RNA"]]$counts

# Scale all genes
GE <- ScaleData(obj, features = rownames(obj))
cat("Scaling done:", nrow(obj), "genes scaled.\n")
Scaling done: 950 genes scaled.
# Run PCA on all genes
GE <- RunPCA(GE, features = rownames(GE), seed.use = 42)

# Elbow plot: shows variance explained per PC
# Use this to decide how many PCs to retain (look for the "elbow")
ElbowPlot(GE, ndims = 50) +
  ggtitle("Elbow plot — variance explained per PC") +
  theme_minimal()
DimPlot(GE, reduction = "pca", dims = c(1, 2), pt.size = 1, alpha = 0.2) +
  ggtitle("PCA — PC1 vs PC2") + theme_minimal()

UMAP

UMAP is run on the top PCs (not on raw gene expression). It produces a 2D embedding that preserves local neighbourhood structure and is well-suited to visualising discrete cell populations.

Parameters:
dims: number of PCs to use — should match your elbow plot choice.
n.neighbors: size of the local neighbourhood (typical range: 15–50).
min.dist: controls how tightly cells are packed (lower = tighter clusters).

GE <- RunUMAP(GE,
              dims        = 1:50,   # adjust based on elbow plot
              n.neighbors = 30,
              min.dist    = 0.3,
              metric      = "euclidean",
              seed.use    = 42)

DimPlot(GE, reduction = "umap", pt.size = 0.3, alpha = 0.3) +
  ggtitle("UMAP (before clustering)") + theme_minimal()

FindClusters() detects communities in the SNN graph. The key parameter is resolution: higher values produce more, smaller clusters.

Step 5 — Identify marker genes per cluster

To interpret clusters biologically, we identify genes that are specifically highly expressed in each cluster relative to all other cells.

FindAllMarkers() runs a one-vs-rest differential expression test.
only.pos = TRUE returns only up-regulated genes (the most useful for annotation).
min.pct and logfc.threshold filter out weak signals to speed up computation.
⚠️ This step can take 5–10 minutes on the full dataset.

markers <- FindAllMarkers(GE,
                          only.pos       = TRUE,
                          min.pct        = 0.25,
                          logfc.threshold = 0.5,
                          test.use       = "wilcox")

# Top 5 markers per cluster
top5 <- markers %>%
  group_by(cluster) %>%
  slice_max(order_by = avg_log2FC, n = 5)

print(top5)

Step 6 — Automated cell type annotation

We use the celldex Mouse RNA-seq reference to automatically assign a cell type label to each cell. SingleR computes correlations between each query cell and reference cell type profiles.

⚠️ This step can take 10–20 minutes on the full dataset.
Why not ZeiselBrainData()? That function has a known incompatibility with newer versions of dbplyr (collect(Inf) is no longer valid). celldex is the recommended replacement and requires no manual normalisation step.
label.main gives coarse labels (neurons, astrocytes, oligodendrocytes, endothelial cells, microglia, etc.); use label.fine for finer subtypes.

# Install celldex if needed
if (!require("celldex", quietly = TRUE))
  BiocManager::install("celldex")
library(celldex)

# Load the mouse RNA-seq reference (downloads once, cached locally after first run)
ref <- celldex::MouseRNAseqData()
cat("Reference label distribution (label.main):\n")
print(table(ref$label.main))

# Convert Seurat object to SingleCellExperiment (required by SingleR)
GE_sce <- as.SingleCellExperiment(GE)

# Run SingleR: correlate each cell against reference profiles
pred <- SingleR(
  test      = GE_sce,
  ref       = ref,
  labels    = ref$label.main,   # switch to ref$label.fine for finer resolution
  de.method = "wilcox"
)

# Transfer predicted labels back to Seurat object
GE$SingleR_label <- factor(pred$pruned.labels)
cat("\nAnnotation done. Label distribution:\n")
print(table(GE$SingleR_label, useNA = "always"))
